� SSH配置

今天主要配置了SSH密钥，用于Git与GitHub之间的安全通信。

SSH配置工作流

本地电脑 (项目) ←→ GitHub (远程仓库)
↓ ↓
SSH 密钥认证 SSH 密钥验证
↓ ↓
推送代码 (git push) 接收代码
拉取代码 (git pull) 存储代码

1. 生成 SSH 密钥对
# 在 Git Bash 或 PowerShell 中运行
ssh-keygen -t rsa -b 4096 -C "你的GitHub邮箱"

# 一路按回车（使用默认路径，可以不设置密码）
# 生成后会有两个文件：
# ~/.ssh/id_rsa - 私钥（留本地，绝不泄露）
# ~/.ssh/id_rsa.pub - 公钥（上传到 GitHub）
2. 查看并复制公钥
# 查看公钥内容
cat ~/.ssh/id_rsa.pub

# 或 Windows 用
notepad ~/.ssh/id_rsa.pub
3. 添加公钥到 GitHub
  1. 登录 GitHub
  2. 点击右上角头像 → Settings
  3. 左侧菜单 → SSH and GPG keys
  4. 点击 New SSH key
  5. 填写：
    • Title：设备名称（如 My PC、Laptop）
    • Key：粘贴刚才复制的公钥
  6. 点击 Add SSH key
4. 测试 SSH 连接
ssh -T git@github.com

# 成功后会显示：
# Hi 用户名! You've successfully authenticated...
5. 将仓库地址改为 SSH
# 查看当前远程地址
git remote -v

# 删除 HTTPS 地址
git remote remove origin

# 添加 SSH 地址
git remote add origin git@github.com:用户名/仓库名.git

# 确认修改成功
git remote -v
